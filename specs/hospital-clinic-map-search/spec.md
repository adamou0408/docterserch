## 需求摘要

使用者因運動傷害需要就醫，但目前查詢各醫院骨科、復健科、中醫的門診時間需要逐一瀏覽各醫院官方網站，非常耗時且難以比較。希望建立一個以地圖為核心的搜尋平台，讓使用者在地圖上選擇一個位置後，能立即查看周邊醫院與診所的可掛號醫生、門診時間及診別資訊，快速做出就醫決定。

## 使用者故事

### 患者（就醫者）

#### US-1：依地圖位置搜尋周邊醫療院所
- **作為**患者，**我想要**在地圖上選擇一個位置並設定搜尋範圍，**以便**快速找到附近有相關科別的醫院和診所
- **驗收條件**：
  - [ ] 使用者可在地圖上點選任意位置或使用目前定位
  - [ ] 使用者可設定搜尋半徑（如 1km / 3km / 5km / 10km）
  - [ ] 地圖上以標記顯示搜尋範圍內的所有醫療院所
  - [ ] 每個標記顯示醫院名稱與距離
  - [ ] 搜尋結果在 3 秒內回應

#### US-2：依科別篩選醫療院所
- **作為**患者，**我想要**依科別（骨科、復健科、中醫等）篩選醫療院所，**以便**只看到符合我就醫需求的醫院
- **驗收條件**：
  - [ ] 提供科別篩選選項（至少包含：骨科、復健科、中醫、其他常見科別）
  - [ ] 支援同時選擇多個科別
  - [ ] 篩選後地圖標記與列表即時更新
  - [ ] 顯示各醫院具備的已選科別

#### US-3：查看門診時間與醫生資訊
- **作為**患者，**我想要**查看特定醫院的門診時間表與看診醫生資訊，**以便**找到適合我時間的門診並決定掛號
- **驗收條件**：
  - [ ] 點選醫院標記後顯示該院的門診時刻表
  - [ ] 門診時刻表顯示：日期、時段（上午/下午/晚上）、科別、醫生姓名
  - [ ] 可篩選特定日期範圍（如今天、明天、本週）
  - [ ] 標示目前是否仍可掛號（若資料來源支援）
  - [ ] 提供前往該醫院掛號頁面的連結

#### US-4：比較不同醫院的門診選項
- **作為**患者，**我想要**在同一個介面中比較多家醫院的門診時間，**以便**選擇最方便的就醫選項
- **驗收條件**：
  - [ ] 搜尋結果以列表形式同時展示多家醫院的門診摘要
  - [ ] 列表可依距離、最近可掛號時間排序
  - [ ] 每筆結果顯示：醫院名稱、距離、最近可看診時段、科別

### 系統管理員（資料維護者）

#### US-5：管理醫療院所資料來源
- **作為**系統管理員，**我想要**監控與管理醫療院所的資料爬取狀態，**以便**確保門診資訊的正確性與即時性
- **驗收條件**：
  - [ ] 後台顯示各資料來源的最後更新時間
  - [ ] 爬取失敗時發送告警通知
  - [ ] 可手動觸發特定醫院的資料重新爬取
  - [ ] 顯示資料覆蓋率（已爬取 vs 總醫院數）

## 衝突標記
- ⚠️ 患者需要即時掛號狀態（US-3, US-4），但系統管理員受限於爬取頻率與來源穩定性（US-5）→ 見 [conflicts/CONFLICT-001.md](../../conflicts/CONFLICT-001.md)

## 非功能需求
- 效能：地圖搜尋結果應在 3 秒內回應；頁面首次載入時間 < 2 秒
- 相容性：支援主流行動瀏覽器（Chrome, Safari, Firefox）與桌面瀏覽器；RWD 響應式設計
- 可用性：系統可用率 > 99%（不含資料來源爬取失敗的情況）
- 資料新鮮度：門診時刻表資料每日至少更新一次

## 安全性需求
- 資料分類：公開（醫院門診為公開資訊）
- 認證需求：初期不需登入即可查詢；後續若加入個人化功能再評估
- 授權需求：管理後台需管理員權限；一般查詢無需權限
- 加密需求：傳輸中加密（HTTPS）
- 審計日誌：管理員操作需記錄（手動更新資料、重新爬取等）
- 個資處理：不涉及 PII（不蒐集使用者個人資訊）；使用者定位資訊僅在前端使用，不傳送至後端

## 成功指標
- 目標 1：上線後 30 天內，每日活躍使用者達到 100 人
- 目標 2：使用者從開始搜尋到找到可掛號門診的平均時間 < 2 分鐘（對比目前逐一查官網的 15-30 分鐘）
- 目標 3：醫療院所資料覆蓋率達到目標區域的 80% 以上
- 量測方式：Google Analytics 追蹤頁面使用量與操作流程；後台統計資料覆蓋率

## 開放問題（已決策）
- [x] 初期涵蓋的地區範圍？→ **特定縣市先行**，驗證後再擴展
- [x] 門診資料的主要來源？→ **健保署開放資料 + 爬取醫院官網**，健保署為基礎清單，爬蟲取得門診時刻表
- [x] 掛號方式？→ **僅連結導向**，初期提供各醫院掛號頁面連結
- [x] 使用者帳號功能？→ **初期不需要**，MVP 為純查詢工具
- [x] 中醫診所資料來源？→ **與西醫分開處理**，中醫可能需額外資料來源（如中醫師公會）
